# Contributing Guide

## Pull Request Checklist

- Code follows existing module boundaries.
- Tests are added or updated.
- API changes are reflected in documentation.
- Database changes use Flyway migrations.
- No secrets or generated build files are committed.
- Security-sensitive changes include threat considerations.

## Commit Style

Use clear, scoped messages:

```text
feat(api): add audit log dashboard
fix(mobile): handle offline lesson completion
docs: add deployment guide
```
